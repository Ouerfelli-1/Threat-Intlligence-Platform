# auth — Security Model

## Password storage — argon2id

Passwords are hashed with **argon2id** (`argon2-cffi`), the current OWASP
recommendation for password storage. argon2id is memory-hard, resisting
GPU/ASIC brute-forcing in a way bcrypt/PBKDF2 do not.

- Plaintext passwords are never stored or logged.
- The bootstrap admin password (`BOOTSTRAP_ADMIN_PASSWORD`) is hashed at
  seed time; the operator is instructed to rotate it after first login.

## Token signing — RS256 asymmetric

JWTs are signed with **RS256** (RSA + SHA-256), an asymmetric scheme:

- Only auth holds the **private** key (signs).
- Any service can hold the **public** key (verifies) without being able to
  forge tokens.

This asymmetry is what historically allowed every service to validate
tokens locally (no call back to auth per request) — the hot path never
depended on auth being up. After the auth simplification the data services
no longer validate, but the property remains for auth's own endpoints and
for any future re-enablement.

Keys are generated by `infra/bootstrap/seed_secrets.py`, stored in the
vault as `AUTH_RS256_PRIVATE_KEY` / `AUTH_RS256_PUBLIC_KEY`, and loaded at
auth startup via `security.init_keys`.

## Token lifetimes

| Token | TTL | Storage |
|---|---|---|
| Access JWT | 1 hour | client-side only (Zustand persisted) |
| Refresh token | 30 days | SHA-256 hash in `auth.sessions` |

Refresh tokens are stored **hashed** (SHA-256) — a database compromise
does not yield usable refresh tokens.

## Session revocation

JWTs are stateless and cannot be individually revoked by signature alone.
The platform layers a session check:

- Each session is a row in `auth.sessions` with a `revoked` flag.
- `/me` and session-sensitive paths re-check the row.
- Role/permission changes (`PATCH /users/{id}`) revoke all of the user's
  sessions.

The 15-second `/me` poll in the frontend makes revocation effective within
one poll interval (the "demote → logout without refresh" requirement).

## RBAC

Three primary roles seeded by `seed.py`:

| Role | Permissions |
|---|---|
| admin | `*` (wildcard) |
| analyst | all `:read` + analyst-layer `:write` (intelligence/threats/iocs/actors/assets/indicator) |
| viewer | all `:read` only |

Plus a `service` role for service accounts (per-service supplementary
permissions).

Enforcement:
- **Server-side** (authoritative): `tip_auth.require_permission(perm)` on
  every protected endpoint across all services; auth's own admin endpoints
  use `deps.require_admin`.
- **Client-side** (cosmetic): `frontend/src/lib/store.ts` mirrors the
  logic to hide chrome a user can't use — never the authority.

## The bootstrap token

`SECRETS_BOOTSTRAP_TOKEN` (in `.env`) authorises the pre-auth vault fetch.
Service-account bootstrap tokens (`SVC_<NAME>_BOOTSTRAP_TOKEN`) are stored
**hashed** in `service_accounts.bootstrap_token_hash` via
`security.hash_token`, so the auth DB never holds usable service tokens in
the clear.

## Audit trail

`auth.audit_log` records actor, action, target, timestamp, and details for
security-relevant events (logins, user/role changes). This is the
compliance officer's reconstruction surface (see
`01_introduction/stakeholders.md`).

## Threats and mitigations

| Threat | Mitigation |
|---|---|
| Credential brute-force | argon2id memory-hardness; (rate-limiting is a future hardening) |
| Token forgery | RS256 — forging requires the private key, held only by auth |
| Stolen refresh token | stored hashed; revocable via session flag |
| Privilege escalation via stale token | session revocation + 15s poll |
| DB compromise reveals passwords | argon2id hashes only |
| DB compromise reveals refresh tokens | SHA-256 hashes only |
| Production misconfiguration (`DISABLE_AUTH=true`) | startup guard refuses to boot |

## Residual risks

- **No login rate-limiting yet** — a brute-force attempt is slowed by
  argon2 cost but not blocked. Documented in
  `08_security/risk_analysis.md` as a future hardening.
- **`FERNET_KEY` compromise** — yields the whole vault including the RS256
  private key. This is the single most important secret to protect; it is
  the one thing intentionally kept outside the vault (in `.env`).
